# VPN и OpenVPN

VPN создаёт защищённый туннель между клиентом и сетью или между двумя сетями. OpenVPN - один из популярных вариантов VPN.

## Зачем нужен VPN

- удалённый доступ сотрудников в офисную сеть;
- связь филиалов;
- доступ к внутренним сервисам;
- защищённое подключение через интернет.

## Client-to-site

Клиент подключается к офисной сети.

```text
laptop -> VPN server -> office LAN
```

Пример: сотрудник из дома подключается к 1С, файловому серверу или RDP.

## Site-to-site

Связь двух сетей.

```text
office A LAN -> VPN -> office B LAN
```

## Что нужно для доступа в LAN через VPN

- VPN-туннель поднялся.
- Клиент получил VPN-IP.
- Есть route до офисной сети.
- Firewall разрешает трафик.
- На обратной стороне есть маршрут назад.
- DNS выдаёт внутренние имена, если они нужны.

## Частая проблема

VPN подключился, но ресурсов офиса не видно.

Проверить:

```bash
ip route
ping <office_gateway>
ping <internal_server_ip>
nslookup internal-name
```

Windows:

```cmd
route print
ping <internal_server_ip>
nslookup internal-name
```

## OpenVPN файлы

Часто клиент получает файл:

```text
client.ovpn
```

В нём могут быть:

- адрес сервера;
- порт;
- протокол;
- сертификаты;
- routes;
- DNS options.

## OpenVPN service на Linux

Пример проверки:

```bash
systemctl status openvpn
journalctl -u openvpn -f
```

Название сервиса может отличаться:

```bash
systemctl list-units | grep -i openvpn
```

## Что спросить при диагностике VPN

- VPN вообще подключается?
- Какой IP получает клиент?
- Какая офисная подсеть?
- Есть ли маршрут до офисной подсети?
- Работает ли ping по IP?
- Работает ли DNS-имя?
- Не пересекаются ли домашняя и офисная подсети?

## Главное запомнить

- VPN connection не гарантирует доступ ко всем ресурсам.
- После подключения важны routes, firewall, NAT и DNS.
- Пересечение подсетей - частая причина странных проблем.
