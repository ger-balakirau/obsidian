# TCP IP - базовая модель

TCP/IP - это набор правил, по которым устройства общаются в сети. Для системного администратора важно не заучить модель, а уметь диагностировать: есть ли адрес, есть ли шлюз, работает ли DNS, доступен ли порт.

## Что нужно для работы сети

Минимально устройству нужны:

- IP-адрес;
- маска подсети;
- шлюз;
- DNS-сервер.

Пример:

```text
IP:      192.168.1.25
Mask:    255.255.255.0 или /24
Gateway: 192.168.1.1
DNS:     192.168.1.1 или 8.8.8.8
```

## IP-адрес

IP-адрес - адрес устройства в сети.

```text
192.168.1.25
```

Если у двух устройств одинаковый IP, будет конфликт адресов.

## Маска

Маска показывает, какая часть IP относится к сети, а какая к хосту.

```text
192.168.1.25/24
```

Для `/24` сеть обычно:

```text
192.168.1.0
```

А доступные адреса:

```text
192.168.1.1 - 192.168.1.254
```

Подробнее: [[02 - Маска подсети - быстрая памятка]]

## Шлюз

Шлюз нужен, чтобы выйти за пределы своей подсети.

Пример:

```text
ПК 192.168.1.25 хочет попасть на 8.8.8.8
→ адрес не из локальной сети
→ трафик идёт на gateway 192.168.1.1
```

Если шлюз неверный, локальная сеть может работать, а интернет - нет.

## DNS

DNS переводит имя в IP.

```text
ya.ru -> 77.88.55.242
```

Если IP пингуется, а имя нет - часто проблема в DNS.

Подробнее: [[05 - DNS]]

## TCP и UDP

TCP - соединение с контролем доставки.

Примеры:

- SSH;
- HTTP/HTTPS;
- SMTP;
- IMAP;
- FTP.

UDP - проще и быстрее, без такого контроля доставки.

Примеры:

- DNS;
- DHCP;
- некоторые VPN;
- VoIP.

## Порт

Порт помогает понять, к какой службе идёт подключение.

Примеры:

```text
22    SSH
53    DNS
80    HTTP
443   HTTPS
3389  RDP
```

## Базовая проверка

Linux:

```bash
ip a
ip route
ping 8.8.8.8
ping ya.ru
nslookup ya.ru
```

Windows:

```cmd
ipconfig /all
ping 8.8.8.8
ping ya.ru
nslookup ya.ru
```

## Главное запомнить

- IP отвечает за адрес устройства.
- Маска определяет границы локальной сети.
- Gateway нужен для выхода в другие сети.
- DNS нужен для имён.
- Порт указывает на конкретную службу.
