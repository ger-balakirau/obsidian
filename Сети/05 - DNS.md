# DNS

DNS переводит имена в IP-адреса. Для администратора DNS критичен: без него ломаются сайты, домен Active Directory, почта, VPN и многие внутренние сервисы.

## Простыми словами

```text
user вводит example.com
→ компьютер спрашивает DNS
→ DNS отвечает IP-адресом
→ компьютер подключается к IP
```

## Основные типы записей

### A

Имя в IPv4-адрес.

```text
server.example.local -> 192.168.1.10
```

### AAAA

Имя в IPv6-адрес.

### CNAME

Псевдоним на другое имя.

```text
www.example.com -> example.com
```

### MX

Почтовые серверы домена.

### PTR

Обратная запись: IP в имя.

## DNS resolver

Resolver - DNS-сервер, который использует клиент.

Посмотреть на Windows:

```cmd
ipconfig /all
```

Linux:

```bash
resolvectl status
cat /etc/resolv.conf
```

## Проверка DNS

Windows:

```cmd
nslookup ya.ru
nslookup ya.ru 8.8.8.8
ipconfig /flushdns
```

Linux:

```bash
nslookup ya.ru
dig ya.ru
resolvectl query ya.ru
```

## Как отличить DNS-проблему от сетевой

Проверить IP:

```bash
ping 8.8.8.8
```

Проверить имя:

```bash
ping ya.ru
```

Если IP работает, а имя нет - вероятно, проблема DNS.

## DNS и Active Directory

Для AD DNS особенно важен. Клиенты ищут контроллеры домена через DNS-записи.

Если DNS настроен неправильно:

- ПК не входит в домен;
- пользователь не логинится;
- GPO не применяется;
- сетевые ресурсы открываются нестабильно.

## Частые проблемы

- указан внешний DNS вместо доменного;
- DNS-сервер недоступен;
- устаревший cache;
- неверная A-запись;
- нет reverse zone там, где она нужна;
- клиент в VPN получил неправильный DNS.

## Главное запомнить

- DNS переводит имя в IP.
- Если `ping 8.8.8.8` работает, а `ping ya.ru` нет - смотри DNS.
- В домене Windows клиенты должны использовать доменный DNS.
- `nslookup` - базовая команда проверки.
