# NAT

NAT меняет адреса в сетевых пакетах. Чаще всего он нужен, чтобы устройства из частной сети выходили в интернет через один внешний IP.

## Зачем нужен NAT

Внутри сети часто используются частные адреса:

```text
192.168.x.x
10.x.x.x
172.16.x.x - 172.31.x.x
```

Они не маршрутизируются напрямую в интернете. Поэтому роутер подменяет внутренний адрес на внешний.

## Пример

```text
ПК:      192.168.1.25
Router:  192.168.1.1
WAN IP:  203.0.113.10
```

Когда ПК идёт в интернет:

```text
192.168.1.25 -> router -> 203.0.113.10 -> internet
```

Ответ возвращается на роутер, а роутер понимает, какому внутреннему устройству его отдать.

## Source NAT

Source NAT меняет исходный адрес.

Типичный случай:

```text
LAN -> Internet
```

## Destination NAT / Port forwarding

Destination NAT меняет адрес назначения.

Типичный случай: проброс порта внутрь сети.

```text
203.0.113.10:443 -> 192.168.1.50:443
```

## NAT и VPN

VPN может подключаться, но доступ в LAN не работать из-за:

- нет маршрута;
- нет NAT;
- firewall блокирует;
- пересекаются подсети;
- DNS не выдан клиенту.

## NAT и troubleshooting

Если из LAN нет интернета:

1. Проверить IP клиента.
2. Проверить gateway.
3. Проверить доступность роутера.
4. Проверить DNS.
5. Проверить NAT на роутере/firewall.
6. Проверить правила firewall.

## Главное запомнить

- NAT позволяет частным сетям выходить наружу.
- Port forwarding - это разновидность destination NAT.
- NAT не заменяет маршрутизацию.
- В VPN проблемы часто связаны с routes + NAT + firewall.
