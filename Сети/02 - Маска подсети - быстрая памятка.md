# Маска подсети - быстрая памятка

## 1. Через CIDR (/n)

- **n** — количество бит сети
- Всего адресов: `2^(32 - n)`
- Доступных: `-2` (сеть и broadcast)

**Примеры:**

- /24 → 256 (254 usable)
- /26 → 64 (62 usable)

## 2. Через маску (255.x.x.x)

| Значение | Биты |
| -------- | ---- |
| 255      | 8    |
| 254      | 7    |
| 252      | 6    |
| 248      | 5    |
| 240      | 4    |
| 224      | 3    |
| 192      | 2    |
| 128      | 1    |
| 0        | 0    |

👉 Складываешь биты → получаешь `/n`

**Пример:**  
255.255.255.192 → 24 + 2 = **/26**

---

## 3. Шаг подсети

Шаг подсети считается по первому октету маски, который не равен `255` и не равен `0`.

Формула:

```text
шаг = 256 - значение октета маски
```

| Октет маски | Биты | Шаг подсети |
| ----------: | ---: | ----------: |
| 254         | 7    | 2           |
| 252         | 6    | 4           |
| 248         | 5    | 8           |
| 240         | 4    | 16          |
| 224         | 3    | 32          |
| 192         | 2    | 64          |
| 128         | 1    | 128         |

Примеры:

```text
255.255.255.252
шаг = 256 - 252 = 4
подсети: .0, .4, .8, .12, .16 ...
```

```text
255.255.255.248
шаг = 256 - 248 = 8
подсети: .0, .8, .16, .24, .32 ...
```

```text
255.255.255.240
шаг = 256 - 240 = 16
подсети: .0, .16, .32, .48, .64 ...
```

Если маска `255.255.254.0`, шаг считается в третьем октете:

```text
шаг = 256 - 254 = 2
подсети: 192.168.0.0, 192.168.2.0, 192.168.4.0 ...
```

---

## ⚡ Быстрый алгоритм

- Есть `/n` → считаешь адреса через формулу
- Есть `255.x.x.x` → считаешь биты → получаешь `/n`
- Нужно найти шаг подсети → `256 - октет маски`
