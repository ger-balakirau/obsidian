# Сетевая диагностика

Сетевая диагностика идёт от простого к сложному: адрес, шлюз, DNS, маршрут, порт, сервис.

## Базовый алгоритм

1. Есть ли IP.
2. Есть ли gateway.
3. Доступен ли gateway.
4. Доступен ли внешний IP.
5. Работает ли DNS.
6. Доходит ли маршрут.
7. Открыт ли порт.
8. Работает ли сервис.

## Linux команды

```bash
ip a
ip route
ping 192.168.1.1
ping 8.8.8.8
ping ya.ru
nslookup ya.ru
traceroute ya.ru
ss -tulnp
nc -vz host 443
```

## Windows команды

```cmd
ipconfig /all
ping 192.168.1.1
ping 8.8.8.8
ping ya.ru
nslookup ya.ru
tracert ya.ru
route print
```

Проверка порта в PowerShell:

```powershell
Test-NetConnection host -Port 443
```

## Если нет IP

Проверить:

- кабель или Wi-Fi;
- VLAN;
- DHCP;
- драйвер сетевой карты;
- не выключен ли интерфейс.

Windows:

```cmd
ipconfig /release
ipconfig /renew
```

## Если есть IP, но нет интернета

Проверить gateway:

```bash
ip route
ping <gateway>
```

Windows:

```cmd
ipconfig /all
ping <gateway>
```

## Если IP пингуется, а имя нет

Проблема похожа на DNS.

```bash
nslookup ya.ru
```

Windows:

```cmd
ipconfig /flushdns
nslookup ya.ru
```

## Если сайт не открывается

Проверить:

```bash
ping site.ru
nslookup site.ru
nc -vz site.ru 443
curl -I https://site.ru
```

## Если сервис недоступен

На сервере:

```bash
ss -tulnp
systemctl status service
journalctl -u service -n 100
```

С клиента:

```bash
nc -vz server port
```

## Главное запомнить

- `ping IP` проверяет сетевую доступность.
- `nslookup name` проверяет DNS.
- `traceroute/tracert` показывает маршрут.
- `nc` или `Test-NetConnection` проверяет порт.
- Если порт открыт, но приложение не работает - смотри сервис и логи.
