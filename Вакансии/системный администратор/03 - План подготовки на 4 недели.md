# План подготовки на 4 недели

План рассчитан на практическую подготовку под вакансию. Лучше идти не только чтением, а через лабораторные работы.

## Неделя 1 - сеть и рабочие места

Цель: уверенно диагностировать типовые проблемы пользователя и сети.

Темы:

- IP, маска, шлюз, DNS.
- DHCP.
- ICMP, ping, tracert/traceroute.
- Windows network settings.
- Общие папки и доступы.
- Helpdesk-подход.

Практика:

- настроить статический IP;
- проверить DHCP lease;
- найти, почему нет интернета;
- проверить DNS-резолвинг;
- подключить сетевую папку;
- составить чеклист “пользователь не видит сеть”.

## Неделя 2 - Windows Server и AD

Цель: понимать доменную инфраструктуру.

Темы:

- Windows Server роли.
- Active Directory.
- OU, users, groups.
- Join computer to domain.
- DNS в домене.
- GPO.
- WSUS.

Практика:

- поднять тестовый Windows Server;
- создать домен;
- завести пользователя;
- ввести ПК в домен;
- создать GPO;
- проверить `gpupdate` и `gpresult`;
- посмотреть Event Viewer.

## Неделя 3 - backup, RAID, VMware

Цель: понимать серверное железо и сохранность данных.

Темы:

- RAID 0/1/10/5/6.
- Supermicro / IPMI базово.
- VMware ESXi / vSphere.
- Snapshots.
- Veeam basics.
- Backup vs snapshot.
- Restore.

Практика:

- нарисовать схему RAID-уровней;
- описать, что делать при degraded RAID;
- создать VM;
- сделать snapshot и удалить его;
- составить backup policy;
- сделать тест восстановления.

## Неделя 4 - Linux, OpenVPN, мониторинг, 1С

Цель: закрыть дополнительные требования и подготовиться к вопросам.

Темы:

- Debian basics.
- SSH.
- systemd.
- OpenVPN.
- Zabbix.
- 1С basics.
- IP-камеры / СКУД overview.

Практика:

- поднять SSH на Debian;
- посмотреть service через `systemctl`;
- посмотреть логи через `journalctl`;
- описать схему OpenVPN;
- добавить host в Zabbix;
- составить список вопросов по 1С и видеонаблюдению.

## Минимальный результат

К концу подготовки нужно уметь уверенно объяснить:

- как работает домен;
- как пользователь получает IP;
- как компьютер находит DNS;
- как применяется GPO;
- как проверить сервис;
- как найти проблему по логам;
- как устроен backup;
- чем отличаются RAID 1, 5, 10;
- что делать, если сервер или сервис недоступен.
