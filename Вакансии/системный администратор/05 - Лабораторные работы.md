# Лабораторные работы

## Лаба 1 - диагностика сети на клиенте

Цель: научиться быстро понимать, почему ПК не видит сеть.

Проверить:

```cmd
ipconfig /all
ping 127.0.0.1
ping <gateway>
ping 8.8.8.8
nslookup ya.ru
tracert ya.ru
```

Результат: чеклист “нет интернета на ПК”.

## Лаба 2 - DHCP и статический IP

Цель: понять разницу между DHCP и ручной настройкой.

Сделать:

- посмотреть текущий IP;
- освободить lease;
- обновить lease;
- настроить статический IP;
- вернуть DHCP.

Команды:

```cmd
ipconfig /release
ipconfig /renew
ipconfig /all
```

## Лаба 3 - Active Directory test lab

Цель: поднять базовый домен.

Сделать:

- создать домен;
- создать OU;
- создать пользователя;
- ввести клиентский ПК в домен.

Результат: схема домена и краткий конспект шагов.

## Лаба 4 - GPO

Цель: понять применение групповых политик.

Сделать:

- создать GPO;
- привязать к OU;
- настроить простую политику;
- применить на клиенте.

Команды:

```cmd
gpupdate /force
gpresult /r
```

## Лаба 5 - Windows Server DNS

Цель: понять DNS в домене.

Сделать:

- создать A-запись;
- проверить резолвинг;
- проверить reverse lookup, если настроен.

Команды:

```cmd
nslookup server.domain.local
ipconfig /flushdns
```

## Лаба 6 - Debian + SSH + systemd

## Лаба 10 - RAID на бумаге

Цель: уметь объяснить RAID без заучивания.

Сделать таблицу:

| RAID | Минимум дисков | Надёжность | Ёмкость | Где использовать |
|---|---:|---|---|---|
| 0 | 2 | нет | сумма дисков | временные данные |
| 1 | 2 | 1 диск может умереть | 50% | простые серверы |
| 5 | 3 | 1 диск может умереть | N-1 | файловые хранилища |
