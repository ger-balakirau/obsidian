# Что такое Git

**Git** - это система контроля версий. Она хранит историю изменений в проекте и позволяет возвращаться к прошлым состояниям, сравнивать версии, работать в ветках и синхронизировать код с другими людьми.

## Простыми словами

Git отвечает на вопросы:

- что изменилось;
- кто изменил;
- когда изменил;
- зачем изменил;
- как вернуться назад, если что-то сломалось.

## Основные сущности

### Working tree

Текущие файлы проекта на диске.

### Staging area

Место, куда ты кладёшь изменения перед commit.

```bash
git add file.txt
```

### Commit

Зафиксированное состояние изменений с сообщением.

```bash
git commit -m "Add config example"
```

### Branch

Отдельная линия работы.

```bash
git switch -c feature/login
```

### Remote

Удалённый репозиторий, например на GitHub или GitLab.

```bash
git remote -v
```

## Как выглядит обычный цикл

```text
изменил файлы
→ посмотрел git status
→ выбрал изменения через git add
→ сделал git commit
→ отправил через git push
```

## Git и GitHub - не одно и то же

- Git - инструмент контроля версий.
- GitHub - сервис для хранения Git-репозиториев и совместной работы.

## Главное запомнить

Git не просто “сохраняет файлы”. Git сохраняет историю решений.
