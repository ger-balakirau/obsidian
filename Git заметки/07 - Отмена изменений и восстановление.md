# Отмена изменений и восстановление

В Git важно сначала понять, где находится изменение: в working tree, staging area или уже в commit.

## Убрать файл из staging

Файл останется изменённым, но не будет готов к commit.

```bash
git restore --staged file.txt
```

## Отменить изменения в файле

Осторожно: команда вернёт файл к последнему commit.

```bash
git restore file.txt
```

## Отменить все локальные изменения

Осторожно: это удалит незакоммиченные изменения.

```bash
git restore .
```

## Исправить последний commit

Если забыл добавить файл:

```bash
git add forgotten-file.txt
git commit --amend
```

Если нужно поменять сообщение:

```bash
git commit --amend -m "New message"
```

## Убрать последний commit, но оставить изменения

```bash
git reset --soft HEAD~1
```

## Убрать последний commit и оставить изменения без staging

```bash
git reset --mixed HEAD~1
```

Это стандартное поведение `reset`, поэтому можно короче:

```bash
git reset HEAD~1
```

Разница:

- `--soft` оставляет изменения в staging;
- `--mixed` оставляет изменения в working tree, но убирает из staging.

## Жёстко откатить working tree

Осторожно: `--hard` удаляет незакоммиченные изменения в отслеживаемых файлах.

Вернуться к последнему commit:

```bash
git reset --hard HEAD
```

Откатиться к конкретному commit:

```bash
git reset --hard <commit>
```

Перед этим проверь:

```bash
git status
git diff
```

Если нужно сохранить изменения перед `--hard`, используй [[09 - Stash - временно спрятать изменения]]:

```bash
git stash push -m "before hard reset"
git reset --hard HEAD
```

## Удалить untracked files

`reset --hard` не удаляет untracked files - файлы, которые Git ещё не отслеживает.

Посмотреть, что будет удалено:

```bash
git clean -fdn
```

Удалить:

```bash
git clean -fd
```

Осторожно: `git clean -fd` удаляет неотслеживаемые файлы и директории.

## Создать обратный commit

Безопасный способ отменить уже опубликованный commit:

```bash
git revert <commit>
```

## Посмотреть commit

```bash
git show <commit>
```

## Найти потерянный commit

Если случайно сделал `reset`, посмотри [[10 - Reflog - поиск потерянных commits]]:

```bash
git reflog
```

## Главное запомнить

- `restore --staged` убирает из staging.
- `restore file` откатывает файл в working tree.
- `reset --soft` двигает историю локально и оставляет изменения.
- `reset --mixed` оставляет изменения, но убирает их из staging.
- `reset --hard` удаляет локальные изменения в отслеживаемых файлах.
- `git clean -fd` удаляет untracked files.
- `revert` безопаснее для уже опубликованной истории.
