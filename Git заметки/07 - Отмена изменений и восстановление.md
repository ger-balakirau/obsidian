# Отмена изменений и восстановление

В Git важно сначала понять, где находится изменение: в working tree, staging area или уже в commit.

## Убрать файл из staging

Файл останется изменённым, но не будет готов к commit.

```bash
git restore --staged file.txt
```

## Отменить изменения в файле

Осторожно: команда вернёт файл к последнему commit.

```bash
git restore file.txt
```

## Отменить все локальные изменения

Осторожно: это удалит незакоммиченные изменения.

```bash
git restore .
```

## Исправить последний commit

Если забыл добавить файл:

```bash
git add forgotten-file.txt
git commit --amend
```

Если нужно поменять сообщение:

```bash
git commit --amend -m "New message"
```

## Убрать последний commit, но оставить изменения

```bash
git reset --soft HEAD~1
```

## Создать обратный commit

Безопасный способ отменить уже опубликованный commit:

```bash
git revert <commit>
```

## Посмотреть commit

```bash
git show <commit>
```

## Главное запомнить

- `restore --staged` убирает из staging.
- `restore file` откатывает файл в working tree.
- `reset --soft` двигает историю локально и оставляет изменения.
- `revert` безопаснее для уже опубликованной истории.
