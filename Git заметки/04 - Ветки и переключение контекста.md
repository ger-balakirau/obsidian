# Ветки и переключение контекста

Ветка - это отдельная линия работы. Обычно `main` или `master` держат стабильной, а новые задачи делают в отдельных ветках.

## Посмотреть ветки

```bash
git branch
```

Все ветки, включая remote:

```bash
git branch -a
```

## Создать новую ветку

```bash
git switch -c feature/login
```

Старый вариант:

```bash
git checkout -b feature/login
```

## Переключиться на ветку

```bash
git switch main
```

## Слить ветку

Перейти в ветку, куда нужно влить изменения:

```bash
git switch main
```

Влить:

```bash
git merge feature/login
```

## Удалить ветку

```bash
git branch -d feature/login
```

Принудительно:

```bash
git branch -D feature/login
```

## Посмотреть, с чем связана ветка

```bash
git branch -vv
```

## Частый рабочий сценарий

```bash
git switch main
git pull
git switch -c feature/new-page
# работаешь с файлами
git add .
git commit -m "Add new page"
git push -u origin feature/new-page
```

## Главное запомнить

- Новую задачу удобно делать в новой ветке.
- Перед созданием ветки обычно обновляют `main`.
- `git switch` меняет текущую ветку.
- `git merge` вливает изменения одной ветки в другую.
