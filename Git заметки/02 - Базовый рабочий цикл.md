# Базовый рабочий цикл Git

Эта заметка про повседневный цикл: проверить состояние, добавить изменения, сделать commit и отправить на remote.

## 1. Проверить состояние

```bash
git status
```

Показывает:

- какие файлы изменены;
- какие файлы добавлены в staging area;
- какие файлы не отслеживаются;
- на какой ты ветке.

## 2. Посмотреть изменения

```bash
git diff
```

Посмотреть изменения, уже добавленные в staging:

```bash
git diff --staged
```

## 3. Добавить изменения в staging

Один файл:

```bash
git add file.txt
```

Папку:

```bash
git add src/
```

Все изменения:

```bash
git add .
```

## 4. Сделать commit

```bash
git commit -m "Add user settings"
```

Хороший commit message коротко говорит, что изменилось.

## 5. Посмотреть историю

```bash
git log --oneline
```

С графом:

```bash
git log --oneline --graph --decorate --all
```

## 6. Отправить изменения

```bash
git push
```

Если ветка новая:

```bash
git push -u origin feature/name
```

## Типовой цикл целиком

```bash
git status
git diff
git add .
git diff --staged
git commit -m "Describe change"
git push
```

## Главное запомнить

- `git status` запускай часто.
- `git add` выбирает, что попадёт в commit.
- `git commit` фиксирует изменения локально.
- `git push` отправляет commits на remote.
