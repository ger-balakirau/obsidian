## Что должно стать понятно

- Чем отличаются working tree, staging area и commit.
- Зачем нужен `git status`.
- Почему `git add` и `git commit` - разные шаги.
- Чем отличаются локальная и удалённая ветка.
- Когда использовать `pull`, `push`, `fetch`.
- Как безопасно отменять изменения.
- Чем отличаются `reset --soft`, `reset --mixed` и `reset --hard`.
- Как временно спрятать изменения через `stash`.
- Как найти потерянный commit через `reflog`.

## Минимум на память

```bash
git status
git add file
git commit -m "message"
git log --oneline
git branch
git switch branch
git stash
git reflog
git pull
git push
```

## Главная мысль

Git хранит историю проекта. Хорошая работа с Git - это маленькие осмысленные commits, понятные ветки и аккуратная синхронизация с remote.
