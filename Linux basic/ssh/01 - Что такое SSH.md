
**SSH** — это способ безопасно подключаться к другому компьютеру или серверу через интернет или локальную сеть.

Расшифровывается как **Secure Shell** — «безопасная оболочка».

SSH чаще всего используют, чтобы:

- управлять сервером удалённо;
- запускать команды на Linux-сервере;
- загружать и скачивать файлы;
- подключаться к GitHub, GitLab и другим сервисам;
- настраивать сайты, приложения и базы данных.

### Простыми словами

SSH позволяет открыть терминал удалённого компьютера так, будто ты сидишь прямо за ним.

Например:

```bash
ssh user@server.com
```

Здесь:

```text
ssh
```

— команда для подключения,

```text
user
```

— имя пользователя на сервере,

```text
server.com
```

— адрес сервера.

### Почему SSH безопасный

SSH шифрует соединение между твоим компьютером и сервером. Это значит, что посторонние не могут просто прочитать логин, пароль или команды, которые ты отправляешь.

### Как обычно подключаются по SSH

Есть два основных способа:

**1. По паролю**

```bash
ssh user@server.com
```

После этого сервер попросит пароль.

**2. По SSH-ключу**

SSH-ключ — это более безопасный способ входа без постоянного ввода пароля.

Обычно есть два файла:

```text
id_rsa      — приватный ключ
id_rsa.pub  — публичный ключ
```

Или современный вариант:

```text
id_ed25519      — приватный ключ
id_ed25519.pub  — публичный ключ
```

Публичный ключ можно передавать и добавлять на сервер.  
Приватный ключ нельзя никому показывать.

### Где используется SSH

SSH часто нужен при работе с:

- VPS и выделенными серверами;
- Linux;
- Docker;
- Git;
- деплоем сайтов;
- облачными серверами;
- системным администрированием.

### Главное запомнить

SSH — это безопасный способ удалённо подключаться к серверу и управлять им через терминал. Самый надёжный вариант подключения — через SSH-ключи.
