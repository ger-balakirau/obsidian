Публичный ключ нужно добавить на сервер в файл пользователя, под которым будет вход:

```text
~/.ssh/authorized_keys
```

Для пользователя `deploy` это обычно:

```text
/home/deploy/.ssh/authorized_keys
```

---

## Вариант 1 — через ssh-copy-id

Работает только если у пользователя уже есть вход по паролю или ключу.

```bash
ssh-copy-id deploy@server.com
```

С нестандартным портом:

```bash
ssh-copy-id -p 2222 deploy@server.com
```

С указанием публичного ключа, который добавляем:

```bash
ssh-copy-id -i ~/.ssh/id_ed25519.pub deploy@server.com
```

С указанием публичного ключа и порта:

```bash
ssh-copy-id -i ~/.ssh/id_ed25519.pub -p 2222 deploy@server.com
```

С указанием ключа, которым заходим на сервер:

```bash
ssh-copy-id -o IdentityFile=~/.ssh/admin_key deploy@server.com
```

С указанием ключа входа и порта:

```bash
ssh-copy-id -o IdentityFile=~/.ssh/admin_key -p 2222 deploy@server.com
```

С указанием ключа, который добавляем, ключа входа и порта:

```bash
ssh-copy-id -i ~/.ssh/id_ed25519.pub -o IdentityFile=~/.ssh/admin_key -p 2222 deploy@server.com
```

Если у `deploy` ещё нет ни пароля, ни ключа — `ssh-copy-id deploy@server.com` не подойдёт. Нужно заходить через `root` или другого пользователя с `sudo`.

---

## Вариант 2 — вручную через root

На сервере, куда добавляешь доступ, зайти под `root`:

```bash
ssh root@server.com
```

С портом:

```bash
ssh -p 2222 root@server.com
```

С ключом входа:

```bash
ssh -i ~/.ssh/root_key root@server.com
```

С ключом входа и портом:

```bash
ssh -i ~/.ssh/root_key -p 2222 root@server.com
```

На целевом сервере выполнить:

```bash
mkdir -p /home/deploy/.ssh
nano /home/deploy/.ssh/authorized_keys
```

Вставить публичный ключ, затем:

```bash
chown -R deploy:deploy /home/deploy/.ssh
chmod 700 /home/deploy/.ssh
chmod 600 /home/deploy/.ssh/authorized_keys
```

## Вариант 3 — одной командой через root

С сервера/компьютера, где лежит публичный ключ:

```bash
cat ~/.ssh/id_ed25519.pub | ssh root@server.com "mkdir -p /home/deploy/.ssh && cat >> /home/deploy/.ssh/authorized_keys && chown -R deploy:deploy /home/deploy/.ssh && chmod 700 /home/deploy/.ssh && chmod 600 /home/deploy/.ssh/authorized_keys"
```

С нестандартным портом:

```bash
cat ~/.ssh/id_ed25519.pub | ssh -p 2222 root@server.com "mkdir -p /home/deploy/.ssh && cat >> /home/deploy/.ssh/authorized_keys && chown -R deploy:deploy /home/deploy/.ssh && chmod 700 /home/deploy/.ssh && chmod 600 /home/deploy/.ssh/authorized_keys"
```

С ключом, которым заходим на сервер:

```bash
cat ~/.ssh/id_ed25519.pub | ssh -i ~/.ssh/root_key root@server.com "mkdir -p /home/deploy/.ssh && cat >> /home/deploy/.ssh/authorized_keys && chown -R deploy:deploy /home/deploy/.ssh && chmod 700 /home/deploy/.ssh && chmod 600 /home/deploy/.ssh/authorized_keys"
```

С ключом входа и нестандартным портом:

```bash
cat ~/.ssh/id_ed25519.pub | ssh -i ~/.ssh/root_key -p 2222 root@server.com "mkdir -p /home/deploy/.ssh && cat >> /home/deploy/.ssh/authorized_keys && chown -R deploy:deploy /home/deploy/.ssh && chmod 700 /home/deploy/.ssh && chmod 600 /home/deploy/.ssh/authorized_keys"
```

Где:

```text
~/.ssh/id_ed25519.pub
```

— публичный ключ, который добавляем пользователю `deploy`.

```text
~/.ssh/root_key
```

— приватный ключ, которым уже заходим на сервер под `root`.

## Вариант 4 — одной командой через пользователя с sudo

Стандартный порт:

```bash
cat ~/.ssh/id_ed25519.pub | ssh user@server.com "sudo mkdir -p /home/deploy/.ssh && sudo tee -a /home/deploy/.ssh/authorized_keys > /dev/null && sudo chown -R deploy:deploy /home/deploy/.ssh && sudo chmod 700 /home/deploy/.ssh && sudo chmod 600 /home/deploy/.ssh/authorized_keys"
```

Нестандартный порт:

```bash
cat ~/.ssh/id_ed25519.pub | ssh -p 2222 user@server.com "sudo mkdir -p /home/deploy/.ssh && sudo tee -a /home/deploy/.ssh/authorized_keys > /dev/null && sudo chown -R deploy:deploy /home/deploy/.ssh && sudo chmod 700 /home/deploy/.ssh && sudo chmod 600 /home/deploy/.ssh/authorized_keys"
```

С ключом входа:

```bash
cat ~/.ssh/id_ed25519.pub | ssh -i ~/.ssh/admin_key user@server.com "sudo mkdir -p /home/deploy/.ssh && sudo tee -a /home/deploy/.ssh/authorized_keys > /dev/null && sudo chown -R deploy:deploy /home/deploy/.ssh && sudo chmod 700 /home/deploy/.ssh && sudo chmod 600 /home/deploy/.ssh/authorized_keys"
```

С ключом входа и нестандартным портом:

```bash
cat ~/.ssh/id_ed25519.pub | ssh -i ~/.ssh/admin_key -p 2222 user@server.com "sudo mkdir -p /home/deploy/.ssh && sudo tee -a /home/deploy/.ssh/authorized_keys > /dev/null && sudo chown -R deploy:deploy /home/deploy/.ssh && sudo chmod 700 /home/deploy/.ssh && sudo chmod 600 /home/deploy/.ssh/authorized_keys"
```

Где:

```text
user@server.com
```

— пользователь, у которого уже есть доступ и `sudo`.

```text
~/.ssh/admin_key
```

— приватный ключ для входа этим пользователем.

## Если вход по ключу не работает

На сервере проверить SSH-конфиг:

```bash
nano /etc/ssh/sshd_config
```

Должно быть:

```text
PubkeyAuthentication yes
AuthorizedKeysFile .ssh/authorized_keys
```

Перезапустить SSH:

```bash
systemctl restart ssh
```

Если служба называется `sshd`:

```bash
systemctl restart sshd
```

Проверить права ещё раз:

```bash
chown -R deploy:deploy /home/deploy/.ssh
chmod 700 /home/deploy/.ssh
chmod 600 /home/deploy/.ssh/authorized_keys
```

Проверить логин подробно:

```bash
ssh -v -i ~/.ssh/id_ed25519 -p 2222 deploy@server.com
```
