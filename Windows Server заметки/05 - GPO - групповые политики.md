# GPO - групповые политики

GPO управляет настройками пользователей и компьютеров в домене.

## Что можно делать через GPO

- настраивать парольные политики;
- маппить сетевые диски;
- задавать настройки firewall;
- устанавливать скрипты входа;
- ограничивать Control Panel;
- разворачивать ярлыки;
- задавать настройки браузера;
- применять security settings.

## Где управлять

```text
Group Policy Management
```

## Куда привязывается GPO

GPO можно связать с:

- site;
- domain;
- OU.

Чаще всего GPO привязывают к OU.

## Computer и User Configuration

```text
Computer Configuration - настройки компьютера
User Configuration     - настройки пользователя
```

Если политика лежит в Computer Configuration, она применяется к компьютерам.

Если в User Configuration - к пользователям.

## Принудительно применить политики

На клиенте:

```cmd
gpupdate /force
```

## Проверить, что применилось

```cmd
gpresult /r
```

HTML-отчёт:

```cmd
gpresult /h report.html
```

## Почему GPO не применяется

Частые причины:

- объект не в той OU;
- GPO не linked;
- security filtering не пускает;
- WMI filter не подходит;
- проблема DNS;
- клиент не видит контроллер домена;
- политика настроена в User, а ожидают на Computer;
- политика конфликтует с другой GPO.

## Порядок диагностики

1. Проверить OU объекта.
2. Проверить link GPO.
3. Проверить security filtering.
4. Запустить `gpupdate /force`.
5. Проверить `gpresult /r`.
6. Проверить DNS и доступность DC.

## Главное запомнить

- GPO применяются к пользователям и компьютерам.
- OU важна для применения политик.
- `gpupdate /force` применяет политики.
- `gpresult /r` показывает результат.
