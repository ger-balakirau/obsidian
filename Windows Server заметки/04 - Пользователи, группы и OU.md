# Пользователи, группы и OU

Эта тема про базовую организацию объектов в Active Directory.

Практика с сервисной учёткой для ввода ПК в домен: [[17 - Настройка пользователя joinuser для ввода ПК в домен]].

## Пользователь

User account - учётная запись человека или сервисная учётная запись.

Обычно у пользователя есть:

- login;
- display name;
- password;
- groups;
- attributes;
- enabled/disabled state.

## Компьютер

Computer account появляется, когда ПК вводят в домен.

Пример:

```text
PC-BUH-01
LAPTOP-IVANOV
```

## OU

OU помогает организовать структуру.

Пример:

```text
OU=Users
OU=Workstations
OU=Servers
OU=Disabled
```

Зачем OU:

- порядок в AD;
- применение GPO;
- делегирование администрирования.

Пример делегирования: дать `joinuser` право создавать computer objects только в `OU=Workstations`, не делая его доменным администратором.

## Группы

Группы нужны для прав и рассылок.

Типовая логика доступа:

```text
User -> Security Group -> Resource Permission
```

Пример:

```text
ivanov -> GG_Finance -> доступ к \\fs01\finance
```

## Создать пользователя

GUI:

```text
Active Directory Users and Computers
```

PowerShell:

```powershell
New-ADUser -Name "Ivan Ivanov" -SamAccountName ivanov -Enabled $true
```

## Добавить пользователя в группу

```powershell
Add-ADGroupMember -Identity "GG_Finance" -Members ivanov
```

Проверить группы:

```cmd
whoami /groups
```

## Отключить пользователя

Лучше отключать, а не удалять сразу.

```powershell
Disable-ADAccount -Identity ivanov
```

## Главное запомнить

- OU - структура и GPO.
- Группы - права доступа.
- Пользователей лучше не наделять правами напрямую.
- Сервисным учёткам нужно выдавать минимум прав под конкретную задачу.
- Уволенных пользователей обычно сначала disable, потом уже архивируют/удаляют по процедуре.
