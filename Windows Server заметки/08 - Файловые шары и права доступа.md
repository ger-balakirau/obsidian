# Файловые шары и права доступа

Файловая шара - сетевой доступ к папке Windows Server.

## UNC path

Сетевой путь обычно выглядит так:

```text
\\server\share
```

Пример:

```text
\\fs01\finance
```

## Share permissions и NTFS permissions

Есть два слоя прав:

```text
Share permissions
NTFS permissions
```

Итоговый доступ - самый ограничивающий из двух.

## Хорошая практика

На share часто ставят широкое разрешение, а детальные права задают через NTFS.

Пример:

```text
Share: Everyone / Full Control
NTFS:  GG_Finance / Modify
```

В реальной инфраструктуре правила могут отличаться, но идея такая: не путать два слоя прав.

## Доступ через группы

Правильно:

```text
user -> group -> folder permission
```

Плохо:

```text
user -> folder permission напрямую
```

## Проверка доступа

На клиенте:

```cmd
whoami
whoami /groups
net use
```

Подключить сетевой диск:

```cmd
net use Z: \\fs01\finance
```

Отключить:

```cmd
net use Z: /delete
```

## Частые причины отказа

- пользователь не в нужной группе;
- Kerberos/логон-токен ещё не обновился;
- NTFS запрещает доступ;
- share permissions запрещают доступ;
- неверный путь;
- DNS не резолвит имя сервера;
- сервер недоступен;
- SMB заблокирован firewall.

## Главное запомнить

- `\\server\share` - сетевой путь.
- Есть share-права и NTFS-права.
- Доступ лучше выдавать через группы.
- После добавления в группу пользователю часто нужно перелогиниться.
