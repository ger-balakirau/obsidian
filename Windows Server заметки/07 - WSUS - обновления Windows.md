# WSUS - обновления Windows

WSUS централизованно управляет обновлениями Windows в организации.

## Зачем нужен WSUS

- не качать обновления на каждом ПК напрямую из интернета;
- контролировать, какие обновления устанавливаются;
- группировать компьютеры;
- откладывать проблемные обновления;
- видеть статус установки.

## Основная модель

```text
Microsoft Update -> WSUS server -> clients
```

Клиенты получают настройки WSUS обычно через GPO.

## Группы компьютеров

Пример:

```text
Test
Workstations
Servers
```

Хорошая практика: сначала проверять обновления на тестовой группе.

## Что настраивается в GPO

Путь:

```text
Computer Configuration
Administrative Templates
Windows Components
Windows Update
```

Важные параметры:

- адрес WSUS-сервера;
- режим установки;
- расписание;
- запрет авто-reboot в рабочее время.

## Проверка клиента

```cmd
gpupdate /force
gpresult /r
```

Проверка событий:

```text
Event Viewer -> Applications and Services Logs -> Microsoft -> Windows -> WindowsUpdateClient
```

## Частые проблемы

- GPO не применился;
- клиент смотрит не на тот WSUS;
- проблемы DNS;
- WSUS давно не синхронизировался;
- не хватает места на диске WSUS;
- обновление не approved;
- клиент не попал в нужную группу.

## Главное запомнить

- WSUS управляет обновлениями централизованно.
- Клиентов обычно направляют на WSUS через GPO.
- Обновления лучше сначала тестировать на небольшой группе.
