# Автозапуск и зависимости

Systemd умеет запускать сервисы при старте системы и учитывать зависимости между units.

## Enable и start - разные вещи

Запустить сейчас:

```bash
sudo systemctl start nginx
```

Включить автозапуск:

```bash
sudo systemctl enable nginx
```

Включить автозапуск и сразу запустить:

```bash
sudo systemctl enable --now nginx
```

## Disable и stop - тоже разные вещи

Остановить сейчас:

```bash
sudo systemctl stop nginx
```

Отключить автозапуск:

```bash
sudo systemctl disable nginx
```

## Проверить автозапуск

```bash
systemctl is-enabled nginx
```

## Targets

`target` - группа units, похожая на “режим системы”.

Частый target для серверов:

```text
multi-user.target
```

В unit-файле:

```ini
[Install]
WantedBy=multi-user.target
```

Это значит: сервис будет запускаться при обычном многопользовательском режиме.

## After и Requires

`After` задаёт порядок запуска:

```ini
After=network.target
```

`Requires` задаёт жёсткую зависимость:

```ini
Requires=postgresql.service
```

Частая связка:

```ini
[Unit]
After=network.target postgresql.service
Requires=postgresql.service
```

## Посмотреть зависимости

```bash
systemctl list-dependencies nginx
```

Обратные зависимости:

```bash
systemctl list-dependencies --reverse nginx
```

## Главное запомнить

- `start` - запустить сейчас.
- `enable` - включить запуск при старте системы.
- `After` - порядок.
- `Requires` - зависимость.
- `WantedBy=multi-user.target` - типичный автозапуск для серверного сервиса.
