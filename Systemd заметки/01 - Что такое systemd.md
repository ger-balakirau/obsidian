# Что такое systemd

`systemd` - это система инициализации и менеджер сервисов в большинстве современных Linux-дистрибутивов. Он запускает систему, управляет сервисами, следит за зависимостями и собирает логи через journal.

## Простыми словами

`systemd` отвечает за вопросы:

- какие сервисы запустить при старте системы;
- работает ли сервис сейчас;
- как перезапустить сервис;
- почему сервис упал;
- какие логи связаны с сервисом.

## Unit

Unit - это объект, которым управляет systemd.

Частые типы:

```text
.service  сервис
.timer    запуск по расписанию
.target   группа состояний системы
.socket   socket activation
.mount    mount point
```

## Service

Service unit описывает процесс или демон:

```text
nginx.service
ssh.service
docker.service
```

Обычно `.service` можно не писать:

```bash
systemctl status nginx
```

То же самое:

```bash
systemctl status nginx.service
```

## Где используется

- запуск веб-сервера;
- автозапуск приложения;
- просмотр логов;
- диагностика падений;
- periodic tasks через timers;
- управление зависимостями между сервисами.

## Главное запомнить

`systemd` - центральное место управления сервисами в Linux. Главные инструменты: `systemctl` и `journalctl`.
