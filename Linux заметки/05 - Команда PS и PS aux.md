



## Что важно освоить
Работа и поиск процесов в PS
## 1. 

```bash
ps #покажет процессы текущего терминала
ps -p # проверяет конкретный процесс
ps -pf # -f показывает расширенную информацию о процессе.
ps aux #покажет процессы всех пользователе
ps aux | grep "[b]ash" #что бы не нашло само себя
ps aux --sort=-%mem | grep "[b]ash"
ps aux --sort=-%cpu
pgrep bash #показать PID bash
pgrep -l bash # показать вместе с именем
pgrep -a bash #-a показывает PID и полную командную строку процесса.
pgrep -n bash # -n означает newest — показать только самый новый процесс из найденных.
pgrep -o bash # -o означает oldest — показать самый старый процесс.
pgrep -af bash -f   # искать по полной командной строке (в строке есть слово bash)
```

### Ключи

```bash
a   # процессы всех пользователей, связанные с терминалами
u   # подробный user-oriented формат
x   # процессы без управляющего терминала
--sort=-%mem #сортирует по памяти от большего к меньшему.
--sort=%mem #сортирует по памяти от меньшего к большему
# Важно: сортировку лучше делать до grep, чтобы ps сортировал весь список процессов, а потом мы оставляли только bash.
```

### часто используют именно так

```bash
ps aux | grep nginx
ps aux | grep ssh
ps aux | grep python
```

## 2 Kill pkill

```bash
kill 12345 # kill без указания сигнала отправляет обычный сигнал завершения SIGTERM (мягко коректно)
kill -9 12345 # Без коректного завершения жосткоьпринудительно.
pkill sleep # все процессы по имени
killall sleep # альтернативный вариант
pkill -9 sleep # жостка
pkill -KILL sleep # более читаемо
```

## 3 запуск процессов
```bash
sleep 100 & # & в конце запускает команду в фоне, и терминал сразу возвращает приглашение командной строки. Пример: [1] 12345. Где 12345 — PID фонового процесса.

```

```bash
jobs  # показывает задачи, которые текущий shell запустил в фоне или остановил.
$! # PID последнего процесса, запущенного в фоне.
PID=$! #сохраняет PID последнего фонового процесса в переменную PID.
sleep 1 # подождать 1 секунду
|| # выполняеь команду следующую если предыдущая была с ошибкой.
&& # вополняет команду после удачного выполнения предыдущей
; # выполняет next команду
sleep 1000 & PID=$! && kill "$PID" && sleep 1 && { ps -p "$PID" || echo stopped; } # {} экранируют команды туту  echo stopped зависит только от ps -p ; закрыать нужно 
```

```bash
$(command)   # результат выполнения команды
$variable    # значение переменной
!!           # последняя команда
!$           # последний аргумент прошлой команды
!?text?      # последняя команда, содержащая text
```
