## ВВОД

В Linux у каждого файла и директории есть:

```text
владелец
группа
права доступа
```

Посмотреть это можно так:

```bash
ls -l
```

Пример вывода:

```bash
-rw-r--r-- 1 dima developers 1200 file.txt
drwxr-xr-x 2 dima developers 4096 project
```

Разберём строку:

```text
-rw-r--r-- 1 dima developers 1200 file.txt
```

```text
-           тип объекта
rw-r--r--   права доступа
dima        владелец
developers  группа
file.txt    имя файла
```

---

## Тип объекта

Первый символ показывает тип:

```text
-  обычный файл
d  директория
l  символическая ссылка
```

Примеры:

```bash
-rw-r--r-- file.txt
drwxr-xr-x project
lrwxrwxrwx link -> file.txt
```

---

## Пользователь и группа

У файла есть владелец:

```text
dima
```

И группа:

```text
developers
```

Пример:

```bash
-rw-r--r-- 1 dima developers file.txt
```

Это значит:

```text
владелец файла — dima
группа файла — developers
```

Узнать текущего пользователя:

```bash
whoami
```

Узнать свои группы:

```bash
groups
```

Более подробно:

```bash
id
```

Пример:

```bash
uid=1000(dima) gid=1000(dima) groups=1000(dima),1001(developers)
```

---

## Посмотреть всех пользователей

Локальные пользователи хранятся в `/etc/passwd`.

```bash
cat /etc/passwd
```

Но чаще удобнее использовать `getent`, потому что он учитывает не только локальные файлы, но и другие источники пользователей, если они настроены:

```bash
getent passwd
```

Показать только имена пользователей:

```bash
cut -d: -f1 /etc/passwd
```

Через `getent`:

```bash
getent passwd | cut -d: -f1
```

Показать обычных пользователей с UID от 1000:

```bash
awk -F: '$3 >= 1000 && $3 < 65534 {print $1, $3, $6, $7}' /etc/passwd
```

Поля в `/etc/passwd`:

```text
user:x:uid:gid:comment:home:shell
```

Пример:

```text
dima:x:1000:1000:Dima:/home/dima:/bin/bash
```

## Посмотреть все группы

Локальные группы:

```bash
cat /etc/group
```

Через `getent`:

```bash
getent group
```

Показать только имена групп:

```bash
cut -d: -f1 /etc/group
```

Поля в `/etc/group`:

```text
group:x:gid:members
```

Пример:

```text
developers:x:1001:dima,anna
```

## Посмотреть группы конкретного пользователя

Коротко:

```bash
groups dima
```

Подробно:

```bash
id dima
```

Пример:

```bash
uid=1000(dima) gid=1000(dima) groups=1000(dima),27(sudo),1001(developers)
```

## Посмотреть всех пользователей и их группы

Простой вариант для обычных локальных пользователей:

```bash
for u in $(awk -F: '$3 >= 1000 && $3 < 65534 {print $1}' /etc/passwd); do
  echo "=== $u ==="
  id "$u"
done
```

Более коротко:

```bash
for u in $(getent passwd | awk -F: '$3 >= 1000 && $3 < 65534 {print $1}'); do
  groups "$u"
done
```

Если нужно увидеть всех, включая системных:

```bash
for u in $(getent passwd | cut -d: -f1); do
  groups "$u"
done
```

Обычно системных пользователей очень много, поэтому для админской проверки чаще фильтруют UID от `1000`.

## Права доступа

Права делятся на три группы:

```text
user   group   others
```

Или проще:

```text
владелец   группа   остальные
```

Пример:

```bash
-rw-r--r--
```

Разделяем на части:

```text
rw-   r--   r--
```

```text
rw-  права владельца
r--  права группы
r--  права остальных
```

---

## Что значат r, w, x

Для файла:

```text
r = можно читать файл
w = можно изменять файл
x = можно запускать файл как программу
```

Пример:

```bash
-rw-r--r--
```

Значит:

```text
владелец может читать и изменять
группа может только читать
остальные могут только читать
```

---

## Примеры прав

```bash
-rw-r--r--
```

Обычный файл, права `644`.

```text
владелец: читать, писать
группа: читать
остальные: читать
```

```bash
-rwxr-xr-x
```

Исполняемый файл или скрипт, права `755`.

```text
владелец: читать, писать, запускать
группа: читать, запускать
остальные: читать, запускать
```

```bash
-rw-------
```

Файл доступен только владельцу, права `600`.

```text
владелец: читать, писать
группа: нет прав
остальные: нет прав
```

---

## Права у директорий

У директорий права работают немного иначе.

Для директории:

```text
r = можно посмотреть список файлов внутри
w = можно создавать, удалять и переименовывать файлы внутри
x = можно войти в директорию
```

Пример:

```bash
drwxr-xr-x project
```

Значит:

```text
владелец может входить, смотреть, создавать и удалять
группа может входить и смотреть
остальные могут входить и смотреть
```

Важно:

```text
Чтобы зайти в директорию через cd, нужен x.
Чтобы увидеть список файлов через ls, нужен r.
Чтобы создавать или удалять файлы внутри, нужен w и x.
```

---

## Числовая форма прав

Права часто записывают числами.

```text
r = 4
w = 2
x = 1
```

Складываем:

```text
rwx = 4 + 2 + 1 = 7
rw- = 4 + 2 = 6
r-x = 4 + 1 = 5
r-- = 4
--- = 0
```

Примеры:

```text
755 = rwxr-xr-x
644 = rw-r--r--
600 = rw-------
700 = rwx------
```

## chmod — изменить права

`chmod` меняет права доступа.

Пример:

```bash
chmod 644 file.txt
```

Результат:

```bash
-rw-r--r-- file.txt
```

Пример:

```bash
chmod 755 script.sh
```

Результат:

```bash
-rwxr-xr-x script.sh
```

Добавить право на запуск:

```bash
chmod +x script.sh
```

Убрать право на запуск:

```bash
chmod -x script.sh
```

Добавить право записи группе:

```bash
chmod g+w file.txt
```

Убрать право записи у остальных:

```bash
chmod o-w file.txt
```

Обозначения:

```text
u = user, владелец
g = group, группа
o = others, остальные
a = all, все
```

---

## chown — изменить владельца

`chown` меняет владельца файла.

```bash
sudo chown dima file.txt
```

Можно сразу изменить владельца и группу:

```bash
sudo chown dima:developers file.txt
```

Для директории со всем содержимым:

```bash
sudo chown -R dima:developers project
```

`-R` значит рекурсивно, то есть применить ко всему внутри.

---

## chgrp — изменить группу

`chgrp` меняет только группу.

```bash
sudo chgrp developers file.txt
```

Для директории со всем содержимым:

```bash
sudo chgrp -R developers project
```

## Частые права

Для обычного файла:

```bash
chmod 644 file.txt
```

Для приватного файла:

```bash
chmod 600 secrets.txt
```

Для скрипта:

```bash
chmod 755 script.sh
```

или:

```bash
chmod +x script.sh
```

Для директории:

```bash
chmod 755 project
```

Для приватной директории:

```bash
chmod 700 private
```

---

## Важная мысль

Права проверяются так:

```text
1. Если ты владелец файла — используются права user.
2. Если ты не владелец, но входишь в группу файла — используются права group.
3. Если не подошло первое и второе — используются права others.
```

Linux не складывает все права вместе.  
Он выбирает только одну подходящую категорию.

Пример:

```bash
----rwx--- 1 dima developers file.txt
```

Если пользователь `dima` является владельцем, то для него используются первые права:

```text
---
```

Даже если группа имеет:

```text
rwx
```

Владелец всё равно не сможет читать файл, потому что его категория — `user`.

## Мини-шпаргалка

```bash
whoami              # кто я
id                  # мой uid, gid и группы
groups              # мои группы

ls -l               # посмотреть права
chmod 644 file      # изменить права
chmod +x script.sh  # сделать файл исполняемым

chown user file     # изменить владельца
chown user:group file
chgrp group file    # изменить группу
```

Главное запомнить:

```text
user  group  others
r     w      x
4     2      1
```

И самые частые варианты:

```text
644 = обычный файл
755 = программа или директория
600 = приватный файл
700 = приватная директория
```
