## Что должно стать понятно

- Чем отличаются владелец, группа и остальные.
- Что значат `r`, `w`, `x` у файла.
- Почему `x` у директории означает “можно пройти внутрь”.
- Когда использовать `chmod`, `chown`, `chgrp`.
- Почему SSH и сервисы часто ломаются из-за прав.
- Чем отличаются hard link и symlink.
- Когда нужны `umask`, ACL, `setgid` и `sticky bit`.

## Минимум для практики

```bash
whoami
id
groups
ls -la
ls -ld dir
namei -l /path/to/file
chmod 644 file
chmod 755 dir
chown user:group file
```

## Частая логика диагностики

Если “permission denied”:

1. Проверить пользователя: `whoami`, `id`.
2. Проверить файл: `ls -l file`.
3. Проверить директорию: `ls -ld dir`.
4. Проверить весь путь: `namei -l /full/path/to/file`.
5. Если есть `+` в `ls -l`, посмотреть ACL: `getfacl file`.
