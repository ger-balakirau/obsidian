

## Пользователь, группы, права

Кто я:

```bash
whoami
```

Показать uid, gid и группы:

```bash
id
```

Показать группы текущего пользователя:

```bash
groups
```

Показать права файлов и директорий:

```bash
ls -l
```

Показать права самой директории, а не содержимого:

```bash
ls -ld dir
```

Показать скрытые файлы тоже:

```bash
ls -la
```

---

## chmod — изменить права

Задать права числом:

```bash
chmod 644 file.txt
```

Обычный файл:

```bash
chmod 644 file.txt
```

Приватный файл:

```bash
chmod 600 secret.txt
```

Исполняемый файл или скрипт:

```bash
chmod 755 script.sh
```

Приватная директория:

```bash
chmod 700 private
```

Обычная директория:

```bash
chmod 755 project
```

Директория для работы группы:

```bash
chmod 775 project
```

Добавить запуск файлу:

```bash
chmod +x script.sh
```

Убрать запуск:

```bash
chmod -x script.sh
```

Добавить право записи группе:

```bash
chmod g+w file.txt
```

Убрать право записи у остальных:

```bash
chmod o-w file.txt
```

Обозначения:

```text
u = user, владелец
g = group, группа
o = others, остальные
a = all, все
```

---

## chown — изменить владельца

Изменить владельца файла:

```bash
sudo chown dima file.txt
```

Изменить владельца и группу:

```bash
sudo chown dima:developers file.txt
```

Изменить владельца и группу у директории со всем содержимым:

```bash
sudo chown -R dima:developers project
```

`-R` значит рекурсивно, то есть применить ко всему внутри.

---

## chgrp — изменить группу

Изменить группу файла:

```bash
sudo chgrp developers file.txt
```

Изменить группу директории:

```bash
sudo chgrp developers project
```

Изменить группу директории со всем содержимым:

```bash
sudo chgrp -R developers project
```

# Особые права

## setuid

Поставить `setuid`:

```bash
chmod u+s program
```

То же самое числом:

```bash
chmod 4755 program
```

Убрать `setuid`:

```bash
chmod u-s program
```

Посмотреть пример системного файла с `setuid`:

```bash
ls -l /usr/bin/passwd
```

`setuid` значит:

```text
файл запускается с правами владельца файла
```

---

## setgid

Поставить `setgid` на файл или директорию:

```bash
chmod g+s file_or_dir
```

То же самое числом:

```bash
chmod 2755 program
```

Убрать `setgid`:

```bash
chmod g-s file_or_dir
```

`setgid` на файле значит:

```text
файл запускается с правами группы файла
```

`setgid` на директории значит:

```text
новые файлы и папки внутри наследуют группу директории
```

---

## setgid для общей директории

Создать директорию проекта:

```bash
sudo mkdir /srv/project
```

Поставить группу:

```bash
sudo chgrp developers /srv/project
```

Поставить права с `setgid`:

```bash
sudo chmod 2775 /srv/project
```

Проверить:

```bash
ls -ld /srv/project
```

Пример результата:

```bash
drwxrwsr-x 2 root developers /srv/project
```

Буква `s` у группы показывает `setgid`.

---

## sticky bit

Поставить `sticky bit`:

```bash
chmod +t shared
```

То же самое числом:

```bash
chmod 1777 shared
```

Убрать `sticky bit`:

```bash
chmod -t shared
```

Проверить `/tmp`:

```bash
ls -ld /tmp
```

`sticky bit` значит:

```text
в общей директории нельзя удалять чужие файлы
```

Пример:

```bash
drwxrwxrwt shared
```

Буква `t` в конце показывает `sticky bit`.

# inode, hard link, symlink

## inode

Показать inode:

```bash
ls -i
```

Показать inode вместе с правами:

```bash
ls -li
```

Показать подробную информацию о файле:

```bash
stat file.txt
```

`inode` — внутренняя запись о файле.

```text
имя файла -> inode -> данные
```

---

## hard link

Создать hard link:

```bash
ln file.txt hard.txt
```

Проверить inode:

```bash
ls -li
```

Если inode одинаковый, это hard links на один файл:

```text
file.txt и hard.txt — два имени одного и того же inode
```

Удалить одно имя:

```bash
rm file.txt
```

Если остался другой hard link, данные не удалятся:

```bash
cat hard.txt
```

---

## symlink

Создать symlink:

```bash
ln -s file.txt link.txt
```

Показать ссылку:

```bash
ls -l
```

Пример:

```bash
link.txt -> file.txt
```

Показать, куда указывает symlink:

```bash
readlink link.txt
```

Удалить symlink:

```bash
rm link.txt
```

Это удалит ссылку, а не файл, на который она указывает.

Создать symlink на директорию:

```bash
ln -s /var/log logs
```

Перейти по ссылке:

```bash
cd logs
```

# umask

Показать текущий `umask`:

```bash
umask
```

Временно задать `umask`:

```bash
umask 022
```

Обычный вариант:

```bash
umask 022
```

Новые права:

```text
file = 644
dir  = 755
```

Для общей работы в группе:

```bash
umask 002
```

Новые права:

```text
file = 664
dir  = 775
```

Приватный вариант:

```bash
umask 077
```

Новые права:

```text
file = 600
dir  = 700
```

`umask` влияет только на новые файлы и директории.

# ACL

## Посмотреть ACL

```bash
getfacl file.txt
```

Посмотреть ACL директории:

```bash
getfacl project
```

Если в `ls -l` есть `+`, значит у файла или директории есть ACL:

```bash
-rw-rw-r--+ file.txt
```

---

## Дать права пользователю

Дать пользователю `anna` чтение и запись:

```bash
setfacl -m u:anna:rw file.txt
```

Дать пользователю `anna` полный доступ к директории:

```bash
setfacl -m u:anna:rwx project
```

---

## Дать права группе

Дать группе `developers` чтение и запись:

```bash
setfacl -m g:developers:rw file.txt
```

Дать группе `developers` полный доступ к директории:

```bash
setfacl -m g:developers:rwx project
```

---

## Default ACL

Задать наследуемые права для группы:

```bash
setfacl -d -m g:developers:rwx project
```

`default ACL` значит:

```text
новые файлы и директории внутри будут получать эти ACL
```

---

## Убрать ACL

Убрать ACL конкретного пользователя:

```bash
setfacl -x u:anna file.txt
```

Убрать ACL конкретной группы:

```bash
setfacl -x g:developers file.txt
```

Убрать все дополнительные ACL:

```bash
setfacl -b file.txt
```

# Общая директория для группы

Задача:

```text
все пользователи группы developers работают в /srv/project
новые файлы получают группу developers
новые файлы получают права для developers
```

Команды:

```bash
sudo mkdir /srv/project
sudo chgrp developers /srv/project
sudo chmod 2775 /srv/project
sudo setfacl -d -m g:developers:rwx /srv/project
```

Проверка:

```bash
ls -ld /srv/project
getfacl /srv/project
```

Ожидаемо:

```bash
drwxrwsr-x+ root developers /srv/project
```

Что важно:

```text
s = setgid, наследование группы
+ = есть ACL
default ACL = наследование прав
```
