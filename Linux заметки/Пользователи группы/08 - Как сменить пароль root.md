# Как сменить пароль root

`root` - главный администратор Linux-системы. Смена его пароля нужна редко, но важно понимать, как это делается и чем отличается от обычной смены пароля пользователя.

## Если есть доступ через sudo

Самый частый и нормальный вариант: ты вошёл обычным пользователем, у которого есть `sudo`.

Сменить пароль `root`:

```bash
sudo passwd root
```

Система попросит:

```text
New password:
Retype new password:
```

Пароль при вводе не отображается. Это нормально.

Проверить вход в root-shell:

```bash
su -
```

Выйти обратно:

```bash
exit
```

## Сменить пароль текущего пользователя

Если нужно сменить пароль того пользователя, под которым ты сейчас вошёл:

```bash
passwd
```

## Сменить пароль другого пользователя

Например, для пользователя `devops`:

```bash
sudo passwd devops
```

## Root может быть заблокирован

В Ubuntu часто прямой вход под `root` отключён. Это не значит, что `sudo` не работает.

Проверить запись root:

```bash
sudo passwd -S root
```

Пример:

```text
root L ...
```

`L` значит locked, пароль заблокирован.

Разблокировать root:

```bash
sudo passwd -u root
```

Заблокировать root обратно:

```bash
sudo passwd -l root
```

На серверах обычно лучше оставить прямой вход под `root` выключенным и работать через обычного пользователя с `sudo`.

## Root-пароль и SSH - разные вещи

Даже если ты задал пароль `root`, вход по SSH под `root` может быть запрещён в `/etc/ssh/sshd_config`.

Параметр:

```text
PermitRootLogin no
```

Это нормальная безопасная настройка.

Если нужно администрировать сервер удалённо:

```text
ssh devops@server
sudo -i
```

Связанная заметка: [[Linux заметки/ssh/16 - Безопасная настройка SSH-сервера]].

## Получить root-shell без смены пароля

Если у пользователя есть `sudo`, можно стать root так:

```bash
sudo -i
```

Или выполнить одну команду от root:

```bash
sudo systemctl restart ssh
```

Часто это лучше, чем включать прямой вход под `root`.

## Если пароль root забыт, но есть физический доступ

Это аварийный сценарий для своей машины или сервера, где у тебя есть законный доступ.

Общая логика:

1. Загрузиться в recovery mode или с Live USB.
2. Получить root-shell.
3. Смонтировать корневую файловую систему в режим записи.
4. Выполнить `passwd root`.
5. Перезагрузиться.

Пример для recovery mode Ubuntu:

```bash
mount -o remount,rw /
passwd root
reboot
```

Если система загружена с Live USB, обычно нужно найти корневой раздел, смонтировать его и зайти через `chroot`:

```bash
lsblk
sudo mount /dev/sda2 /mnt
sudo chroot /mnt
passwd root
exit
sudo reboot
```

`/dev/sda2` - только пример. Сначала всегда смотри реальные разделы через `lsblk`.

## После смены пароля проверить

```bash
su -
whoami
id
```

Ожидаемо:

```text
root
uid=0(root)
```

## Что не забыть

- Не используй простой пароль для `root`.
- Не включай SSH-вход под `root` без необходимости.
- Для ежедневной работы используй обычного пользователя + `sudo`.
- Если меняешь SSH-настройки, сначала проверь конфиг:

```bash
sudo sshd -t
sudo systemctl reload ssh
```

## Минимум на память

```bash
sudo passwd root
passwd
sudo passwd username
sudo -i
su -
sudo passwd -S root
sudo passwd -l root
sudo passwd -u root
```
