## Вводная

В Linux право `x` называется `execute`.

Но у файла и у директории оно означает разные вещи.

```text
x у файла      = можно запустить файл как программу
x у директории = можно войти в директорию и проходить через неё
```

---

## `x` у файла

Для обычного файла `x` означает:

```text
файл можно выполнить
```

Например, есть скрипт:

```bash
ls -l script.sh
```

Результат:

```bash
-rw-r--r-- 1 dima dima script.sh
```

У файла нет `x`.

Попробуем запустить:

```bash
./script.sh
```

Может быть ошибка:

```text
Permission denied
```

Добавляем право на запуск:

```bash
chmod +x script.sh
```

Проверяем:

```bash
ls -l script.sh
```

Результат:

```bash
-rwxr-xr-x 1 dima dima script.sh
```

Теперь файл можно запустить:

```bash
./script.sh
```

---

## Что нужно, чтобы запустить файл

Для запуска файла нужно:

```text
1. У файла должно быть право x.
2. У директории, где лежит файл, должно быть право x.
```

Пример:

```bash
/home/dima/scripts/script.sh
```

Чтобы запустить этот файл, нужно право `x` на директории:

```text
/
 /home
 /home/dima
 /home/dima/scripts
```

И право `x` на самом файле:

```text
script.sh
```

---

## `x` у директории

У директории `x` означает:

```text
можно войти в директорию
можно пройти через директорию дальше
можно обратиться к файлу внутри, если знаешь его имя
```

Пример:

```bash
ls -ld project
```

Результат:

```bash
drwxr-xr-x 2 dima dima project
```

У директории есть `x`, значит в неё можно зайти:

```bash
cd project
```

---

## `r`, `w`, `x` у директории

У файла всё просто:

```text
r = читать файл
w = изменять файл
x = запускать файл
```

У директории иначе:

```text
r = посмотреть список имён внутри
w = создавать, удалять и переименовывать внутри
x = входить и обращаться к объектам внутри
```

---

## Директория с `r`, но без `x`

Пример:

```bash
chmod 644 dir
```

Права:

```bash
drw-r--r-- dir
```

Есть `r`, но нет `x`.

Можно попытаться посмотреть список:

```bash
ls dir
```

Но обычно будет ошибка:

```text
Permission denied
```

Почему?

Потому что `r` даёт право читать список имён,  
но без `x` нельзя нормально зайти внутрь директории и получить доступ к объектам.

Для директорий почти всегда нужен `x`.

---

## Директория с `x`, но без `r`

Пример:

```bash
chmod 711 dir
```

Права:

```bash
drwx--x--x dir
```

У остальных есть `x`, но нет `r`.

Они не могут посмотреть список файлов:

```bash
ls dir
```

Ошибка:

```text
Permission denied
```

Но если они знают точное имя файла внутри, они могут обратиться к нему:

```bash
cat dir/file.txt
```

Это сработает, если у них есть права на сам файл.

---

## Пример: можно открыть файл, но нельзя увидеть список

Есть директория:

```bash
drwx--x--x dir
```

Внутри есть файл:

```bash
-rw-r--r-- file.txt
```

Пользователь не может сделать:

```bash
ls dir
```

Но может сделать:

```bash
cat dir/file.txt
```

Если он знает имя `file.txt`.

Потому что:

```text
x на директории позволяет пройти внутрь
r на файле позволяет читать файл
```

---

## Директория с `w`, но без `x`

Пример:

```bash
chmod 722 dir
```

Права:

```bash
drwx-w--w- dir
```

У остальных есть `w`, но нет `x`.

На практике это почти бесполезно.

Чтобы создать, удалить или переименовать файл внутри директории, обычно нужны оба права:

```text
w + x
```

Только `w` недостаточно.

---

## Чтобы создавать файлы в директории

Нужны права:

```text
w + x на директорию
```

Пример:

```bash
drwxrwxr-x project
```

Группа может создавать файлы внутри, потому что у группы есть:

```text
rwx
```

Создать файл:

```bash
touch project/app.py
```

---

## Чтобы удалить файл из директории

Важно:

```text
удаление файла зависит от прав на директорию,
а не от прав на сам файл
```

Пример:

```bash
drwxrwxr-x project
-r--r--r-- project/file.txt
```

Файл `file.txt` только для чтения.

Но если у пользователя есть `w + x` на директорию `project`, он может удалить файл:

```bash
rm project/file.txt
```

Почему?

Потому что удалить файл — это изменить содержимое директории.  
То есть убрать имя файла из списка директории.

---

## Права на файл и права на директорию работают вместе

Допустим, есть файл:

```bash
/home/dima/project/file.txt
```

Чтобы прочитать файл, нужно:

```text
x на /home
x на /home/dima
x на /home/dima/project
r на file.txt
```

Для чтения файла нужен `r` на файле,  
но для пути к нему нужен `x` на директориях.

---

## Частые варианты прав для директорий

### `755`

```bash
drwxr-xr-x dir
```

Значит:

```text
владелец может всё
группа может входить и смотреть
остальные могут входить и смотреть
```

Часто используется для обычных директорий.

---

### `700`

```bash
drwx------ dir
```

Значит:

```text
только владелец может входить, смотреть и изменять
```

Хорошо для приватных директорий.

---

### `775`

```bash
drwxrwxr-x dir
```

Значит:

```text
владелец и группа могут всё
остальные могут входить и смотреть
```

Удобно для общей рабочей директории группы.

---

### `777`

```bash
drwxrwxrwx dir
```

Значит:

```text
все могут входить, смотреть, создавать и удалять
```

Обычно опасно.

Если нужна общая директория, лучше часто использовать sticky bit:

```bash
chmod 1777 shared
```

Тогда все могут создавать файлы, но не могут удалять чужие.

---

## Частые варианты прав для файлов

### `644`

```bash
-rw-r--r-- file.txt
```

Значит:

```text
владелец может читать и писать
остальные могут только читать
```

Обычный текстовый файл.

---

### `600`

```bash
-rw------- secret.txt
```

Значит:

```text
только владелец может читать и писать
```

Приватный файл.

---

### `755`

```bash
-rwxr-xr-x script.sh
```

Значит:

```text
владелец может читать, писать и запускать
остальные могут читать и запускать
```

Обычный исполняемый файл или скрипт.

---

## Главное отличие

```text
x у файла = разрешение запустить файл
x у директории = разрешение войти в директорию и пройти через неё
```

Без `x` на директории нельзя нормально работать с объектами внутри, даже если у самих файлов есть подходящие права.

---

## Мини-шпаргалка

Для файла:

```text
r = читать содержимое файла
w = изменять содержимое файла
x = запускать файл
```

Для директории:

```text
r = видеть список файлов
w = создавать, удалять, переименовывать
x = входить, проходить, обращаться по имени
```

Команды:

```bash
chmod +x script.sh     # сделать файл исполняемым
chmod 755 dir          # обычная директория
chmod 700 private      # приватная директория
chmod 775 project      # директория для группы
chmod 1777 shared      # общая директория со sticky bit
```

Главное запомнить:

```text
Для доступа к файлу нужны права не только на файл,
но и x на всех директориях в пути к нему.
```
