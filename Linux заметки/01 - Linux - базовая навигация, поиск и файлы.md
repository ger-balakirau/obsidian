# Linux: базовая навигация, поиск и работа с файлами

Эта заметка — первый практический слой. Её цель: научиться уверенно перемещаться по файловой системе, смотреть файлы, искать нужное и соединять команды через пайпы.

## Что важно освоить

- Навигация: `pwd`, `ls`, `cd`.
- Файлы и директории: `touch`, `mkdir`, `cp`, `mv`, `rm`.
- Поиск: `find`, `grep`.
- Просмотр логов и файлов: `cat`, `less`, `head`, `tail`.
- Фильтрация вывода: `|`, `sort`, `uniq`, `wc`, `sed`.

Права доступа здесь даны коротко. Подробный разбор находится в [[Linux заметки/Пользователи группы/00 - Карта темы]].

## 1. Навигация по файловой системе

```bash
pwd                # показать текущую директорию

ls                 # список файлов и папок
ls -l              # подробный список
ls -la             # подробный список + скрытые файлы
ls -lh             # размеры в удобном виде: K, M, G

cd /path/to/dir    # перейти в директорию
cd ..              # на уровень выше
cd ~               # в домашнюю директорию
cd -               # вернуться в предыдущую директорию
```

### Ключи

- `-l` = long format, подробный вывод
- `-a` = all, показать скрытые файлы
- `-h` = human-readable, удобные размеры

## 2. Работа с файлами и папками

```bash
touch file.txt             # создать пустой файл
mkdir dir                  # создать папку
mkdir -p dir/subdir        # создать вложенные папки

cp file1 file2             # копировать файл
cp -r dir1 dir2            # копировать папку с содержимым

mv file1 file2             # переименовать или переместить файл
mv file.txt /tmp/          # переместить файл

rm file.txt                # удалить файл
rm -r dir                  # удалить папку с содержимым
rm -rf dir                 # удалить папку принудительно
```

### Ключи

- `-p` = parents, создать промежуточные папки
- `-r` = recursive, рекурсивно
- `-f` = force, без подтверждения

## 3. Поиск файлов и папок

### По имени

```bash
find /path -name "file.txt"      # найти файл по имени
find /etc -type f -name "*.conf" # найти все .conf файлы
find /var -type d -name "logs"   # найти папки с именем logs
```

### По размеру

```bash
find / -size +100M       # больше 100 MB
find / -size -10M        # меньше 10 MB
```

### По времени изменения

```bash
find / -mtime -1         # изменены меньше 1 дня назад
find / -mtime +7         # изменены больше 7 дней назад
find / -mmin -5          # меньше 5 минут назад
find / -mmin +30         # больше 30 минут назад
find / -newermt "2025-03-01" # Файлы, изменённые после 1 марта 2025 года
find /path -not -newermt "2025-03-01" # Файлы, изменённые до 1 марта 2025 года
find / -newermt "3 months ago" # Файлы за последние 3 месяца
```

### Ключи

- `-name` — искать по имени
- `-type f` — только файлы
- `-type d` — только папки
- `-size` — искать по размеру
- `-mtime` — искать по времени изменения в днях
- `-mmin` искать по времени изменения в минутах
- `-newermt`  "дата" — изменённые после указанной даты
- `-not -newermt` "дата" — изменённые до указанной даты (включая её)
-  `-newermt "N months ago"` — изменённые за последние N месяцев
- `+100M` — больше 100 MB
- `-10M` — меньше 10 MB

## 4. Поиск по содержимому файлов

```bash
grep "text" * 2>/dev/null #без ругани на деректорию файлв в текущей деректории.
grep "text" file.txt             # найти text в файле
ps aux | grep "[b]ash" # что бы не нашло само себя.
grep -l "text" * 2>/dev/null #вывести названия файлов с text
grep -c "text" # вывести количество строк с text
grep -i "text" file.txt          # без учёта регистра
grep -n "text" file.txt          # показать номера строк
grep -v "text" file.txt          # показать строки без совпадения
grep -r "text" /path             # искать рекурсивно в папке
grep -rin "error" /var/log       # искать error в логах
grep "APP" . --exclude=".env" 2>/dev/null # исключает файл .env
grep "APP" . --exclude-dir="logs" 2>/dev/null # исключает деректорию logs

```

### Ключи

- `-r` = recursive, искать по папкам
- `-i` = ignore case, без учёта регистра
- `-n` = number, показат

## 5. Фильтрация и пайпы

### Процессы

```bash
ps aux | grep nginx
```

Показывает все процессы и оставляет только строки с `nginx`.

### Логи ядра

```bash
dmesg | grep -i usb
```

Ищет сообщения про USB.

### Что делает `|`

`|` передаёт вывод одной команды на вход другой.

### Что значит `ps aux`

- `a` — процессы всех пользователей
- `u` — подробный формат
- `x` — процессы без привязки к терминалу

## 6. Сортировка, уникальные строки, подсчёт

```bash
sort file.txt              # отсортировать строки
sort -r file.txt           # сортировка в обратном порядке
uniq file.txt              # убрать соседние дубликаты
sort file.txt | uniq       # отсортировать и убрать дубликаты

wc -l file.txt             # количество строк
wc -w file.txt             # количество слов
wc -c file.txt             # размер в байтах
```

### Ключи

- `-r` = reverse, обратный порядок
- `-l` = lines, строки
- `-w` = words, слова
- `-c` = bytes/chars, байты

---

## 7. Просмотр файлов

```bash
cat file.txt               # показать файл целиком

less file.txt              # удобный просмотр большого файла

head -n 10 file.txt        # начало файла, первые 10 строк
tail -n 10 file.txt        # конец файла, последние 10 строк
tail -f logfile.log        # live-режим, следить за изменениями
```

### Что где

- `head` — начало файла
- `tail` — конец файла
- `tail -f` — live-режим, файл обновляется на экране
- `cat` — сразу весь файл
- `less` — постраничный просмотр

### Ключи

- `-n 10` — показать 10 строк
- `-f` = follow, следить за дописыванием файла

### Полезно в `less`

- `Space` — вниз
- `b` — назад
- `/text` — поиск
- `q` — выход

## 8. Очистка файла, удаление строк, замена слов

### Обнулить файл, не удаляя его

```bash
> file.txt
truncate -s 0 file.txt
```

### Полностью удалить файл

```bash
rm file.txt
```

### Ключи

- `truncate` — изменить размер файла
- `-s 0` — установить размер 0 байт

---

### Удалить определённые строки

```bash
sed '5d' file.txt          # показать файл без 5-й строки
sed -i '5d' file.txt       # удалить 5-ю строку в самом файле

sed -i '5,10d' file.txt    # удалить строки с 5 по 10
sed -i '/error/d' file.txt # удалить все строки с error
```

### Заменить слова

```bash
sed 's/old/new/' file.txt           # заменить первое совпадение в строке
sed 's/old/new/g' file.txt          # заменить все совпадения в строке
sed -i 's/old/new/g' file.txt       # заменить во всём файле
sed -i 's/error/warn/gi' file.txt   # заменить без учёта регистра
```

### Безопасно с backup

```bash
sed -i.bak '/error/d' file.txt
sed -i.bak 's/old/new/g' file.txt
```

### Ключи

- `sed` — потоковый редактор
- `d` = delete, удалить
- `s` = substitute, заменить
- `g` = global, все совпадения
- `i` = ignore case, без учёта регистра
- `-i` = in-place, изменить файл на месте
- `-i.bak` — изменить файл и создать резервную копию

## 9. Права доступа

```bash
ls -l
chmod 755 file
chmod +x script.sh
```

### Пример прав

```bash
-rwxr-xr--
```

### Расшифровка

- первый символ:
    - `-` — обычный файл
    - `d` — директория

Дальше три группы:

- `rwx` — права владельца
- `r-x` — права группы
- `r--` — права остальных

### Значения

- `r` = read, чтение
- `w` = write, запись
- `x` = execute, выполнение

### Что значит `755`

- владелец: `rwx`
- группа: `r-x`
- остальные: `r-x`

## 10. Пользователи и процессы

```bash
whoami                  # текущий пользователь
id                      # UID, GID и группы пользователя

ps aux                  # все процессы
top                     # процессы в реальном времени
kill PID                # корректно завершить процесс
kill -9 PID             # завершить принудительно
```

### Пояснение

- `whoami` — кто сейчас работает в системе
- `id` — информация о пользователе и группах
- `ps aux` — список процессов
- `top` — мониторинг процессов в реальном времени
- `kill` — отправить сигнал процессу

### Поля в `ps aux`

- `USER` — владелец процесса
- `PID` — идентификатор процесса
- `%CPU` — нагрузка на CPU
- `%MEM` — использование памяти
- `COMMAND` — команда запуска

### В `top`

- `q` — выйти
- `k` — завершить процесс
- `M` — сортировка по памяти
- `P` — сортировка по CPU

### Что значит `kill -9`

- `kill PID` — мягкое завершение
- `kill -9 PID` — жёсткое завершение, когда обычное не помогает

## 13 Размер деректории du 

```bash
du -sh logs  #выведет размер папки logs
du -sh * #выведет размер всез файлов и папок в деректории
du -sh .* #выведет размер всез файлов и папок в деректории включая скрытые
du -h --max-depth=1 logs
```

```bash
du    # disk usage
-s     # summary, только итог
-h     # human-readable, удобный формат: K, M, G
*      # все объекты в текущей директории
.*     # включая скрытые
--max-depth=1 #показывает размер самой директории и объектов внутри неё на один уровень глубины.
```

## 15. Короткая шпаргалка по ключам

- `-l` — подробный вывод
- `-a` — скрытые файлы

- `-r` — рекурсивно
- `-f` — принудительно / follow у `tail`

- `-n` — номера строк
- `-v` — исключить совпадения
- `-p` — создать промежуточные папки

- `-type d` — только папки
- `-mtime` — время изменения
