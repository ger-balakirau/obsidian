Файловые дескрипторы — это числовые идентификаторы, которые Linux использует для обращения к открытым файлам/потокам.

Стандартные дескрипторы (есть у каждой программы)

```
0 stdin стандартный ввод клавиатура
1 stdout стандартный вывод экран (терминал)
2 stderr вывод ошибок экран (терминал)
```

Вот короткая памятка по перенаправлению ввода-вывода в bash:

Перенаправление вывода (>, >>)

```bash
>   # перезаписать файл
>>  # дописать в конец файла
```

Перенаправление ошибок (2>)

```bash
2>    # перезаписать stderr (ошибки)
2>>   # дописать stderr в конец файла
&>    # перенаправить stdout и stderr (перезапись)
&>>   # stdout и stderr (дозапись)
```

Перенаправление ввода (<)

```bash
<   # читать из файла вместо клавиатуры
```

Пайпы и связки (|, |&, >&)

```bash
|    # передать stdout одной команды в stdin другой
|&   # передать stdout+stderr (аналог 2>&1 |)
>&   # перенаправить один дескриптор в другой
```

Примеры

```bash
# Ошибки в файл, вывод на экран
ls file fake 2> err.log

# Всё в один файл
ls file fake &> all.log

# Перенаправить stderr в stdout
cmd 2>&1 | grep error

# Оба потока в пайп (bash 4+)
cmd |& grep error

# Чтение из файла
sort < list.txt
```
