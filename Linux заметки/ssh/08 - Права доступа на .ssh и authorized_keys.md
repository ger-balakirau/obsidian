# Права доступа на `.ssh` и `authorized_keys`

SSH специально не доверяет слишком открытым правам. Если директория `.ssh`, файл `authorized_keys` или приватный ключ доступны лишним пользователям, подключение по ключу может не сработать.

## На сервере

```bash
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
```

## На локальном компьютере

```bash
chmod 600 ~/.ssh/id_ed25519
```

Для RSA-ключа:

```bash
chmod 600 ~/.ssh/id_rsa
```

## Проверка

```bash
ls -ld ~/.ssh
ls -l ~/.ssh/authorized_keys
```

## Главное запомнить

- `~/.ssh` на сервере обычно должен быть `700`.
- `~/.ssh/authorized_keys` на сервере обычно должен быть `600`.
- приватный ключ на локальном компьютере обычно должен быть `600`.
