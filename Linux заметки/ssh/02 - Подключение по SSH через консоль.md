
## SSH-подключение выполняется через терминал или консоль.

Базовая команда:

```bash
ssh user@server
```

Примеры:

```bash
ssh root@192.168.1.10
```

```bash
ssh admin@example.com
```

Где:

```text
ssh      — команда подключения
user     — пользователь на сервере
server   — IP-адрес или домен сервера
```

---

## Подключение по стандартному порту

По умолчанию SSH использует порт **22**.

```bash
ssh user@server.com
```

Пример:

```bash
ssh root@185.10.20.30
```

---

## Подключение по нестандартному порту

Если SSH работает не на порту 22, порт указывается через `-p`.

```bash
ssh -p 2222 user@server.com
```

Пример:

```bash
ssh -p 2222 root@185.10.20.30
```

Важно: ключ `-p` пишется с маленькой буквы.

---

## Подключение по паролю

Самый простой вариант подключения:

```bash
ssh user@server.com
```

После выполнения команды сервер попросит пароль:

```text
user@server.com's password:
```

Пароль при вводе обычно не отображается. Это нормально.

---

## Выполнение команды на сервере без входа в сессию

Можно выполнить одну команду на сервере:

```bash
ssh user@server.com "ls -la"
```

Пример:

```bash
ssh root@185.10.20.30 "df -h"
```

Несколько команд:

```bash
ssh user@server.com "cd /var/www && ls -la"
```

---

## Полезные параметры SSH

Подключиться к нестандартному порту:

```bash
ssh -p 2222 user@server.com
```

Показать подробный вывод подключения:

```bash
ssh -v user@server.com
```

Ограничить время ожидания подключения:

```bash
ssh -o ConnectTimeout=10 user@server.com
```

---

## Быстрая шпаргалка

Обычное подключение:

```bash
ssh user@server.com
```

Подключение по другому порту:

```bash
ssh -p 2222 user@server.com
```

Выполнить команду на сервере:

```bash
ssh user@server.com "command"
```

# Главное запомнить

Для обычного подключения достаточно:

```bash
ssh user@server.com
```

Если SSH работает на нестандартном порту:

```bash
ssh -p 2222 user@server.com
```
