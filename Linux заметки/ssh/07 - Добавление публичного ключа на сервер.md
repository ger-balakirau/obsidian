# Добавление публичного SSH-ключа на сервер

Чтобы входить по ключу, публичный ключ нужно добавить на сервер в файл пользователя:

```text
~/.ssh/authorized_keys
```

Для пользователя `deploy` полный путь обычно такой:

```text
/home/deploy/.ssh/authorized_keys
```

Главная мысль: **на сервер добавляется только публичный ключ `.pub`; приватный ключ остаётся на твоём компьютере**.

## Что уже должно быть

- На локальном компьютере есть пара ключей: приватный и публичный.
- На сервере есть пользователь, под которым ты хочешь входить.
- У тебя есть хотя бы один рабочий способ попасть на сервер: пароль, `root` или другой пользователь с `sudo`.

Если ключа ещё нет, смотри [[05 - Генерация SSH-ключа]].

## Вариант 1. Через `ssh-copy-id`

Это самый простой способ, если у пользователя уже разрешён вход по паролю или другому ключу.

```bash
ssh-copy-id deploy@server.com
```

С нестандартным портом:

```bash
ssh-copy-id -p 2222 deploy@server.com
```

Если нужно добавить конкретный публичный ключ:

```bash
ssh-copy-id -i ~/.ssh/id_ed25519.pub deploy@server.com
```

После этого проверь вход:

```bash
ssh deploy@server.com
```

## Вариант 2. Вручную на сервере

Подключись пользователем, у которого уже есть доступ:

```bash
ssh root@server.com
```

Создай директорию и файл для ключей:

```bash
mkdir -p /home/deploy/.ssh
nano /home/deploy/.ssh/authorized_keys
```

Вставь содержимое публичного ключа, например из файла:

```bash
cat ~/.ssh/id_ed25519.pub
```

Затем на сервере выставь владельца и права:

```bash
chown -R deploy:deploy /home/deploy/.ssh
chmod 700 /home/deploy/.ssh
chmod 600 /home/deploy/.ssh/authorized_keys
```

## Вариант 3. Одной командой

С локального компьютера, где лежит публичный ключ:

```bash
cat ~/.ssh/id_ed25519.pub | ssh root@server.com "mkdir -p /home/deploy/.ssh && cat >> /home/deploy/.ssh/authorized_keys && chown -R deploy:deploy /home/deploy/.ssh && chmod 700 /home/deploy/.ssh && chmod 600 /home/deploy/.ssh/authorized_keys"
```

С нестандартным портом:

```bash
cat ~/.ssh/id_ed25519.pub | ssh -p 2222 root@server.com "mkdir -p /home/deploy/.ssh && cat >> /home/deploy/.ssh/authorized_keys && chown -R deploy:deploy /home/deploy/.ssh && chmod 700 /home/deploy/.ssh && chmod 600 /home/deploy/.ssh/authorized_keys"
```

Если заходишь не под `root`, а под пользователя с `sudo`, используй `sudo` внутри команды:

```bash
cat ~/.ssh/id_ed25519.pub | ssh admin@server.com "sudo mkdir -p /home/deploy/.ssh && sudo tee -a /home/deploy/.ssh/authorized_keys > /dev/null && sudo chown -R deploy:deploy /home/deploy/.ssh && sudo chmod 700 /home/deploy/.ssh && sudo chmod 600 /home/deploy/.ssh/authorized_keys"
```

## Проверка

Обычное подключение:

```bash
ssh deploy@server.com
```

С портом:

```bash
ssh -p 2222 deploy@server.com
```

С конкретным приватным ключом:

```bash
ssh -i ~/.ssh/id_ed25519 deploy@server.com
```

С ключом и портом:

```bash
ssh -i ~/.ssh/id_ed25519 -p 2222 deploy@server.com
```

## Если вход по ключу не работает

Проверь на сервере:

```bash
grep -E '^(PubkeyAuthentication|AuthorizedKeysFile)' /etc/ssh/sshd_config
```

Обычно должно быть:

```text
PubkeyAuthentication yes
AuthorizedKeysFile .ssh/authorized_keys
```

Проверь права:

```bash
ls -ld /home/deploy /home/deploy/.ssh
ls -l /home/deploy/.ssh/authorized_keys
```

Исправь права:

```bash
chown -R deploy:deploy /home/deploy/.ssh
chmod 700 /home/deploy/.ssh
chmod 600 /home/deploy/.ssh/authorized_keys
```

Запусти подробную диагностику с локального компьютера:

```bash
ssh -v -i ~/.ssh/id_ed25519 -p 2222 deploy@server.com
```

Подробности: [[15 - Диагностика SSH-подключения]] и [[17 - Частые ошибки SSH]].

## Главное запомнить

- Публичный ключ добавляется в `authorized_keys`.
- Приватный ключ никуда не копируется.
- Файл `authorized_keys` принадлежит тому пользователю, под которым будет вход.
- Неправильные права на `.ssh` часто ломают вход по ключу.
