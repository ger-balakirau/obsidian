# SFTP - интерактивная передача файлов

`sftp` открывает интерактивную файловую сессию поверх SSH. Это удобно, когда нужно посмотреть директории на сервере, вручную загрузить или скачать несколько файлов.

## Подключение

```bash
sftp user@server.com
```

С готовым алиасом:

```bash
sftp work
```

С портом:

```bash
sftp -P 2222 user@server.com
```

С ключом:

```bash
sftp -i ~/.ssh/work_ed25519 user@server.com
```

## Локальная и удалённая стороны

Внутри `sftp` есть две “стороны”:

- удалённая сторона - сервер;
- локальная сторона - твой компьютер.

Команды без `l` работают на сервере:

```text
pwd
ls
cd
mkdir
rm
```

Команды с `l` работают локально:

```text
lpwd
lls
lcd
```

## Основные команды внутри SFTP

```text
pwd        текущая директория на сервере
lpwd       текущая локальная директория
ls         файлы на сервере
lls        локальные файлы
cd dir     перейти в директорию на сервере
lcd dir    перейти в локальную директорию
mkdir dir  создать директорию на сервере
put file   загрузить файл на сервер
get file   скачать файл с сервера
rm file    удалить файл на сервере
exit       выйти
```

## Загрузить файл на сервер

```text
put file.txt
```

Загрузить в конкретное место:

```text
put file.txt /tmp/file.txt
```

Загрузить несколько файлов:

```text
mput *.log
```

## Скачать файл с сервера

```text
get app.log
```

Скачать в конкретное локальное имя:

```text
get /var/log/app.log ./app.log
```

Скачать несколько файлов:

```text
mget *.log
```

## Работа с директориями

Загрузить директорию рекурсивно:

```text
put -r ./site
```

Скачать директорию рекурсивно:

```text
get -r /srv/site
```

## Практический пример

```bash
sftp work
```

Внутри:

```text
lpwd
lcd ./dist
cd /srv/app
put -r .
ls
exit
```

## Когда лучше не SFTP

- Для деплоя проекта лучше `rsync`.
- Для одного файла быстрее `scp`.
- Для повторяемой синхронизации `sftp` неудобен.

## Главное запомнить

- `put` загружает на сервер.
- `get` скачивает с сервера.
- `lcd` меняет локальную директорию.
- `cd` меняет директорию на сервере.
- `sftp work` работает, если настроен `Host work` в `~/.ssh/config`.
