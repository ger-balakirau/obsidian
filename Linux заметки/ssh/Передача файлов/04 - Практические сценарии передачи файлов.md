# Практические сценарии передачи файлов

Эта заметка собирает типовые задачи: загрузить конфиг, скачать лог, задеплоить проект, сделать бэкап или передать файл в директорию, куда нужен `sudo`.

## Загрузить один файл в `/tmp`

```bash
scp file.txt work:/tmp/
```

Проверить:

```bash
ssh work "ls -l /tmp/file.txt"
```

## Загрузить файл в защищённую директорию

Напрямую может не получиться:

```bash
scp nginx.conf work:/etc/nginx/nginx.conf
```

Безопаснее так:

```bash
scp nginx.conf work:/tmp/nginx.conf
ssh work "sudo mv /tmp/nginx.conf /etc/nginx/nginx.conf && sudo nginx -t && sudo systemctl reload nginx"
```

## Скачать лог с сервера

```bash
scp work:/var/log/nginx/error.log ./error.log
```

Или папку логов:

```bash
rsync -av work:/var/log/nginx/ ./nginx-logs/
```

## Задеплоить статический сайт

Сначала dry run:

```bash
rsync -avz --delete --dry-run ./dist/ work:/srv/www/site/
```

Если всё выглядит правильно:

```bash
rsync -avz --delete ./dist/ work:/srv/www/site/
```

## Задеплоить приложение без мусора

```bash
rsync -avz --delete \
  --exclude '.git/' \
  --exclude 'node_modules/' \
  --exclude '.env' \
  ./ work:/srv/app/
```

Перезапустить сервис:

```bash
ssh work "sudo systemctl restart app"
```

## Сделать бэкап директории с сервера

```bash
rsync -avz work:/srv/app/uploads/ ./backups/uploads/
```

С датой в имени папки:

```bash
rsync -avz work:/srv/app/uploads/ "./backups/uploads-$(date +%Y-%m-%d)/"
```

## Передать файл через jump host

Если в [[18 - Каскадный SSH]] настроен `ProxyJump`, можно использовать обычный алиас:

```bash
scp file.txt internal:/tmp/
rsync -av ./project/ internal:/srv/project/
sftp internal
```

Без config:

```bash
scp -o ProxyJump=jump@jump.example.com file.txt deploy@10.0.0.5:/tmp/
```

## Проверить место перед копированием

```bash
ssh work "df -h"
ssh work "du -sh /srv/app"
```

## Проверить права на директорию

```bash
ssh work "ls -ld /srv/app"
ssh work "whoami && id"
```

## Что выбрать

- Один файл: `scp`.
- Папка один раз: `scp -r`.
- Проект или деплой: `rsync`.
- Бэкап с сервера: `rsync`.
- Ручной просмотр и выбор файлов: `sftp`.
- Директория требует `sudo`: копировать в `/tmp`, потом `sudo mv`.
